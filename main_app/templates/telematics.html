Overview

The Telematics Analysis Demo supports these objectives:

  • Display methods to visualize relevant data types expected in Roadtec telematics data
  • Show the ability to understand Roadtec's product portfolio and apply it in useful ways
  • Demonstrate technical skills to rapidly prototype impactful applications
  • Provide a foundation for discussions about skills and requirements for the data analyst position

Telematics Analysis Demo

Technical Overview

Analysis and Visualization Examples

The Telematics Analysis Demo utilizes several types of analysis and visualization methods including:

  • Tabular data lists
  • Time series plots
  • Cross-filtering plots
  • Categorical data analysis
  • Geospatial visualization
Simulated Telematics Dataset

The data used for the Telematics Analysis Demo was generated based on research and experience. It includes an array of data types including:

  • Product Categories (e.g., Asphalt Pavers, Asphalt Screeds, Material Transfer Vehicles)
  • Specific Roadtec Products (e.g., RP-170e/ex, EZ-R2-10, RX-900e/ex, etc.)
  • Representive types of telematics data (e.g., Engine, Screed, GPS, etc.)
  • Corporate data (e.g., customer names, purchase type, product age, etc.)
  • Date, time, and geospatial data

The dataset was created using a Python script and used random distributions to generate representative data sets.

Coding and Database Functions

The Telematics Analysis Demo is built on the Data Analytics app as described on the home page. Specific details are described below:

  • Coding Language. The visualizations are implemented with the Plotly Dash module designed for Python.
  • Database. The simulated data is stored in the MySQL database which supports the Data Analytics app. The simulated data was generated as a CSV file with a Python script. The CSV file was uploaded to the MySQL database using the Dataset Manager function of the Data Analytics app. The Dataset Manager uses the Pandas module to upload CSV data into a new database table in the MySQL database.
  • Data Conditioning. The Pandas module for Python is used to read the data from the MySQL database. Pandas functions are then used to transform the data as needed for the Plotly graphs to include simple parsing of dates and more complex filtering operations on dataframes.

All source code is available on GitHub at https://github.com/TerraX3000/data_analytics_app

Project Plan

The initial version of the Telematics Analysis Demo was focused on developing a foundation of data visualization using representative data. The next steps will focus on applying trending algorithms to the dataset to characterize the data and provide a baseline for anomaly detection.
